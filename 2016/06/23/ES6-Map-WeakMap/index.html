ES6-Map-WeakMap

ECMAScript 6 新增了下面四种数据结构：Map, WeakMap, Set and WeakSet.

Map

  1. Map对象就是简单的键/值映射.其中键和值可以是任意值(原始值或对象值).

  2. Map实例的属性和操作方法示例：

  3. 遍历方法
    keys()：返回键名的遍历器。
    values()：返回键值的遍历器。
    entries()：返回所有成员的遍历器。
    forEach()：遍历Map。

  4. Map结构转为数组结构，比较快速的方法是结合使用扩展运算符（…）,或者 Array.from()。

WeakMap

  1. WeakMap结构与Map结构基本类似，唯一的区别是它只接受对象作为键名（null除外），不接受原始类型
    的值作为键名，当键名是原始值时，会报错：Uncaught TypeError: Invalid value used as weak map key。
    而且键名所指向的对象，不计入垃圾回收机制。set()和get()分别用来添加数据和获取数据:

  2. WeakMap原理图解：

var map = new WeakMap(),
element = document.querySelector(“.element”);

变量map和element各自引用了一个对象.

map.set(element, “Original”);
map中添加了一个键值对,这个键和变量element同时指向一个DOM对象,只是一个是强引用,一个是弱引用.

element.parentNode.removeChild(element);
element = null;
强引用全部主动断掉,最后剩下的弱引用也自动断掉,孤立的DOM元素被回收,WeakMap对象中对应的键值对被自动清除.如果是常规的Map,由于不是弱引用,所以那个DOM对象不会被回收,仍然存在,仍然可以通过get()方法访问到.

所以WeakMap有一个特定的使用情境,那就是在将值映射到一些未来可能被会删除的对象上的时候，比如映射DOM 对象。
在大多数情况下,使用常规的Map应该是你最合适的选择.WeakMap有不少限制,比如不能枚举键值对(for of),也不能得知它们到底包含了多少个键值对(size).
如果你需要这样做,那么就使用常规的Map.如果你仅仅需要把一些对象作为键,且不需要其他更多的功能,那么WeakMap是个好的选择.

总结：Object和Map的比较

  1. 对象的键只能是字符串,但Map的键可以是任意值.
  2. 可以很容易的得到一个Map的键值对个数(map.size),而只能跟踪一个对象的键值对个数
